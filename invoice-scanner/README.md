# 发票识别 Skill v3.0

自动识别所有类型的发票（交通、住宿、餐饮等），提取关键信息并生成分类统计报告和Excel报销表。

## 🎯 功能特性

### 核心功能
- 🔍 **全面识别** - 识别市内交通、长途交通、住宿、餐饮等所有类型发票
- 📂 **智能分类** - 自动按四大类别分类统计
- 📦 **支持 ZIP** - 可以直接处理压缩包
- 🤖 **自动解压** - 自动检测并解压目录中的 ZIP 文件到同级目录
- 📊 **多格式输出** - JSON + Markdown + Excel + CSV
- 🎯 **精准提取** - 提取日期、金额、行程、税务等关键信息
- 💰 **自动统计** - 计算总金额和分类汇总
- 🔄 **智能去重** - 多策略识别重复发票（行程单+电子发票）

### 新增功能（v3.0）
- ✨ **Excel报销表** - 生成标准费用报销明细表（含公司抬头、签字栏）
- 🏷️ **智能重命名** - 自动按规则重命名发票文件（项目-地点-时间-发票号）
- 📅 **日期过滤** - 按日期范围筛选发票
- 🔄 **增量扫描** - 只处理新文件或修改过的文件
- 💱 **多币种支持** - 支持外币发票和汇率转换
- 📋 **数据验证** - 自动检测异常数据（日期、金额、发票号格式）
- 🎯 **置信度评分** - AI识别置信度，低置信度项目自动标注
- 🔐 **PDF密码处理** - 尝试常见密码打开加密PDF
- 📄 **多页PDF** - 智能拆分多页PDF中的多张发票
- 🖼️ **质量检测** - 检测图像模糊、倒置等质量问题
- 🧮 **税务信息** - 提取税率和税额字段
- 🔒 **文件哈希** - SHA256哈希确保文件完整性和去重

## 📖 使用方法

### 基本用法

```bash
# 扫描指定目录
/invoice-scanner ./发票文件夹

# 扫描 ZIP 文件
/invoice-scanner ./receipts.zip

# 扫描当前目录
/invoice-scanner .
```

### 高级用法

```bash
# 生成Excel报销表（含公司抬头）
/invoice-scanner ./发票文件夹 --rename --company "XX科技有限公司" --applicant "张三"

# 按日期范围筛选
/invoice-scanner ./发票文件夹 --from 2025-01-01 --to 2025-12-31

# 增量扫描（只处理新文件）
/invoice-scanner ./发票文件夹 --incremental

# 额外生成CSV格式
/invoice-scanner ./发票文件夹 --csv

# 处理密码保护的PDF
/invoice-scanner ./发票文件夹 --password "123456,888888"

# 组合使用
/invoice-scanner ./发票文件夹 --rename --company "XX公司" --applicant "张三" --from 2025-01-01 --csv
```

## 📋 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `--rename` | 对发票文件进行重命名（交通：项目-起止区间-年月日-发票号；其他：项目-地点-年月日-发票号） | `--rename` |
| `--company <公司名称>` | 设置公司名称（用于Excel表头） | `--company "XX科技有限公司"` |
| `--applicant <报销人>` | 设置报销人姓名（用于Excel表头） | `--applicant "张三"` |
| `--from <日期>` | 起始日期（过滤发票） | `--from 2025-01-01` |
| `--to <日期>` | 结束日期（过滤发票） | `--to 2025-12-31` |
| `--incremental` | 增量扫描模式（只处理新文件） | `--incremental` |
| `--csv` | 额外生成CSV格式报告 | `--csv` |
| `--max-file-size <MB>` | 单个文件最大大小限制（默认50MB） | `--max-file-size 100` |
| `--password <密码>` | PDF密码（多个用逗号分隔） | `--password "123456,888888"` |
| `--rename-separator <分隔符>` | 文件重命名分隔符（默认"——"） | `--rename-separator "-"` |

## 🏷️ 文件重命名规则

启用 `--rename` 参数后，自动按以下规则重命名发票文件：

### 交通发票（火车票、飞机票、出租车、网约车、地铁等）

**命名格式：** `项目——起止区间——年月日——发票号码.扩展名`

**示例：**
- 火车票：`火车票——祁县-太原——2025年11月4日——0329874757558.jpg`
- 火车票：`火车票——太谷西-灵石东——2025年11月10日——0000022397.jpg`
- 飞机票：`飞机票——北京首都-上海虹桥——2025年12月15日——9876543210.pdf`
- 出租车：`出租车——银川站-新百采购——2025年11月12日——28224265.jpg`
- 网约车：`网约车——祁县东-小区——2025年11月18日——0027251600.jpg`
- 地铁：`地铁——未知起点-终点——2025年11月20日——1234567890.jpg`

**字段说明：**
- **项目**：发票类型（火车票、飞机票、出租车、网约车、地铁、公交等）
- **起止区间**：from-to（出发地-目的地）
  - 火车票：出发站-到达站
  - 飞机票：出发机场-到达机场
  - 市内交通：起点-终点
- **年月日**：完整日期（如"2025年11月4日"）
- **发票号码**：取后10-12位

### 其他发票（住宿、餐饮、通讯等）

**命名格式：** `项目——地点——年月日——发票号码.扩展名`

**示例：**
- 住宿：`住宿——兰州——2025年11月12日——437744089.pdf`
- 住宿：`住宿——西安——2025年12月1日——1234567890.pdf`
- 餐饮：`餐饮——西安——2025年9月15日——1234567890.jpg`
- 通讯：`通讯——北京——2025年10月8日——9876543210.pdf`
- 招聘：`招聘——线上——2025年10月16日——1273519934.jpg`

**字段说明：**
- **项目**：发票类型（住宿、餐饮、通讯、办公用品等）
- **地点**：城市或地区名称
  - 住宿：酒店所在城市
  - 餐饮：商户所在城市
  - 其他：商户所在城市
- **年月日**：完整日期（如"2025年11月4日"）
  - 住宿：使用入住日期
  - 其他：使用发票日期
- **发票号码**：取后10-12位

### 特殊情况处理

- 无法提取完整信息的文件，使用原文件名
- 发票号码为空时，使用"无发票号"代替
- 日期不明确时，使用"未知日期"代替
- 交通发票起止区间不明确时，使用"未知起点-终点"
- 其他发票地点不明确时，使用"未知地点"

## 🔍 去重策略

多策略智能去重：

1. **精确重复**：相同文件哈希 → 完全重复，只保留一条
2. **发票号码重复**：相同发票号 + 金额（±0.01元）→ 疑似重复，标记但保留
3. **行程单+发票**：火车票/飞机票的行程单对应电子发票 → 保留发票，行程单标记为重复

## ✅ 数据验证

自动检测以下异常：
- ❌ 日期在未来
- ❌ 金额为负数
- ❌ 发票号码格式异常（长度不足10位）
- ❌ 置信度低于0.7的识别结果

异常项目会标记 `validation_warnings` 字段，便于人工核对。

## ⚙️ 技术原理

使用 Claude 的多模态能力：
1. 读取图片/PDF 文件
2. 图像质量检测（模糊、倒置）
3. AI 视觉分析判断发票类型和分类
4. 提取关键字段信息（含税务信息）
5. 计算文件哈希（SHA256）
6. 置信度评分
7. 多策略去重
8. 数据验证
9. 生成多格式报告

## ⚠️ 注意事项

- 确保发票图片清晰可读（模糊图像会降低识别准确度）
- 支持中文发票（国内航班、火车、打车平台、酒店、餐饮等）
- 自动识别并避免重复发票（文件哈希、发票号码、行程单匹配）
- PDF密码保护：使用 `--password` 参数提供密码，或手动解密后扫描
- 多页PDF：自动检测并拆分独立发票，无法拆分时会标注
- 重命名时智能避免文件名冲突（序号、时间戳、哈希后缀）
- 置信度 < 0.5 的记录不计入统计，需人工核对
- 大批量文件建议使用 `--incremental` 模式分批处理
- 单个文件默认最大50MB，可通过 `--max-file-size` 调整

## 📊 置信度说明

## 🆘 常见问题

### ZIP文件无法解压
```bash
# 检查ZIP文件完整性
unzip -t archive.zip

# 如果是密码保护，先手动解压或使用 --password
```

### 图像识别不准确
- 确保图像清晰、无旋转
- 检查置信度分数（<0.7需人工核对）
- 查看 `quality_warnings` 了解具体问题

### PDF密码保护
```bash
# 提供密码列表
/invoice-scanner ./folder --password "123456,888888,password123"
```

### 文件重命名失败
- 检查文件权限：`ls -la`
- 使用自定义分隔符：`--rename-separator "-"`

### 增量模式不生效
- 首次运行会创建缓存 `.invoice_scanner_cache.json`
- 第二次运行才会跳过已处理文件
- 删除缓存文件可重新全量扫描

### Excel格式异常
- 使用 Microsoft Excel 或 WPS 打开
- CSV文件使用 UTF-8 with BOM 编码
