# 领域地图构建与派别深挖 - 详细流程

这是一份完整的8步工作流程指南，用于将第一轮筛选结果（30-40篇论文）转化为结构化的领域地图。

---

## 开始前的确认

### A1. 确认输入类型

用户提供以下之一或组合：
- **筛选表**：含每篇论文的一句话总结/相关度/是否为综述/派别初判
- **论文清单 + 摘要/引言要点**：已提取的核心信息
- **已有的核心论文列表**：用户手动整理的重要文献

### A2. 输出格式选择

向用户展示以下选项，**必须等待用户选择**后再继续：

| 选项 | 格式 | 适用场景 |
|------|------|----------|
| **A** | Markdown 结构化报告 | 需要完整文档，含小标题与列表 |
| **B** | 对照表优先 | 先看"派别对照表/机会矩阵"，再看解释 |
| **C** | Notion 适配 | 需要页面结构 + 数据库字段建议 + 表格 |
| **D** | 飞书/多维表格适配 | 需要字段建议 + 关键产出表格 |
| **E** | PPT 讲稿大纲风格 | 适合组会汇报，一页一主题的要点 |

如用户说"自定义"，询问：更关心"表格/清单/叙述"哪一种占比最高。

---

## Step 2: 领域地图（仅基于 A+I+C）

**输入**：摘要(A) + 引言(I) + 结论(C) 的信息

**信息来源分步要求**：

| 步骤 | 信息来源 | 理由 | 升级规则 |
|------|----------|------|----------|
| **Step 2** (领域地图) | **仅A/I/C** | 高层次概览，从论文主张推断优缺点 | 如A/I/C不足，标注"[信息不足]" |
| **Step 3** (派别时间线) | **A/I/C + 引言中的相关工作** | 互相批评通常出现在引言的相关工作段落 | 如A/I/C无批评信息，标注"[需相关工作段落]" |
| **Step 6-7** (深读计划) | **全文 OR 模板** | 详细抽取需要完整论文 | 参见3A/3B/3C/3D执行模式 |

**重要提示**：如某步信息不足，明确告知用户需要的信息源，提供空白模板让用户补充。

### 2A. 高频核心论文/方法 Top List

**分层结构**：

| 层级 | 标准 | 入选理由示例 |
|------|------|--------------|
| 必读 | 高被引/奠基性/代表派别综述 | 被引>100，开创性方法，领域综述 |
| 选读 | 重要但非核心/特定场景 | 某派别的改进，特定应用 |
| 了解 | 背景/辅助理解 | 相关工作，早期探索 |

**输出格式**：
```
## 必读论文
1. [作者 年] 标题 - 入选理由：[奠基性工作/高被引/代表XX派别]

## 选读论文
1. [作者 年] 标题 - 入选理由：[XX派别的改进/特定应用场景]
...
```

### 2B. 主要派别划分

**每个派别包含**：
- **名称**：你为该派别命名（如"基于梯度的方法"、"符号推理派"）
- **边界定义**：什么算/不算该派别
- **核心创意一句话**：核心思想是什么
- **代表论文**：3-8篇标志性论文

**输出格式**：
```markdown
### 派别1: [名称]
- **边界定义**：[描述该派别的研究范围和特征]
- **核心创意**：[一句话概括核心思想]
- **代表论文**：
  - [作者1 年1] 标题1
  - [作者2 年2] 标题2
  ...
```

### 2C. 派别特色（优缺点）

**仅基于 A+I+C 支持的信息**：

| 派别 | 主要优点 | 主要缺点 |
|------|----------|----------|
| 派别1 | | |
| 派别2 | | |

**注意**：此时不要引入正文中的详细批评，只基于摘要/引言/结论中明确提到的优缺点。

**输出建议 - 阶段性暂停**：

完成2A/2B/2C后，**向用户确认**：
```
已完成领域概览：
- 2A: 核心论文层级（必读/选读/了解）
- 2B: 识别出 [N] 个派别及代表论文
- 2C: 各派别优缺点简要对比

📊 是否需要继续进行详细派别分析（时间线、互相批评、对照表）？
   [Y] 继续 Step 3 详细分析
   [N] 跳至 Step 4 收敛（当前概览已满足需求）
```

**如用户选择 [Y]**，继续 Step 3；**如用户选择 [N]**，直接跳至 Step 4。

---

## Step 3: 派别 + 时间线归档

**目标**：重读 A+I，按派别分组并按年代排序

### 3.1 构建派别时间线

**每个派别的时间线**：
```
派别X: [起点论文 年] → [关键演化 年] → [近期趋势 年-至今]
```

### 3.2 提取互相批评点

**逐派列出对其他派别的主要批评**：

| 派别 | 被批评对象 | 批评内容 | 来源论文 |
|------|------------|----------|----------|
| 派别A | 派别B | [具体批评] | [论文引用] |

### 3.3 交付：派别对照表

**表格列**：
- 派别名称
- 核心创意
- 关键要求/假设
- 优点
- 缺点
- 代表论文
- 对其他派别的批评点

---

## Step 4: 收敛到核心约20篇 + 选2-3派深挖

### 4.1 修正关键词

**基于核心论文反推**：
- 应增加的关键词
- 应减少的关键词
- 生成的二次检索式（Boolean格式）

**输出格式**：
```
## 关键词调整
- 增加: [关键词1, 关键词2, ...]
- 减少: [关键词1, 关键词2, ...]

## 二次检索式
([关键词1 OR 关键词2] AND [关键词3]) NOT [关键词4]
```

### 4.2 收敛清单：最相关约20篇

**必须给出剔除理由**：
```
## 核心论文列表（20篇）
1. [论文信息] - 保留理由
2. [论文信息] - 保留理由
...

## 剔除论文
- [论文信息] - 剔除理由：[相关度低/质量不高/重复等]
```

### 4.3 深挖选择：选2-3派

**选择理由**（必须包含）：
- 与子题/场景的匹配度
- 成熟度（是否稳定/有争议）
- 潜在突破点
- 学习成本（时间/背景要求）

**输出格式**：
```markdown
## 深挖派别选择

### 选择1: [派别名称]
- **与子题匹配**：[说明]
- **成熟度**：[说明]
- **潜在突破点**：[说明]
- **学习成本**：[说明]
```

---

## Step 5: 抽取评价维度/关键问题

### 5.1 领域关键问题列表

**大家真正关心什么**：
```
Q1: [问题1]
Q2: [问题2]
Q3: [问题3]
...
```

### 5.2 共识评价维度

**常见的优点/缺点维度**：
- 效率 vs 准确性
- 可解释性 vs 性能
- 泛化能力 vs 专精性
- 实施成本 vs 收益
- ...

### 5.3 场景权重

**不同应用场景下哪些维度更重要**：

| 场景 | 高权重维度 | 低权重维度 | 典型示例 |
|------|------------|------------|----------|
| 场景1 | 维度A, 维度B | 维度C | [示例] |
| 场景2 | 维度C, 维度D | 维度A | [示例] |

---

## Step 6-7: 进入正文的深读计划

**注意**：此阶段不是要求替用户把所有正文读完，而是提供"如何读"的执行计划与每篇的抽取模板。

### 6.1 阅读顺序建议

**按以下原则排序**：
- 先读更好懂的（入门/综述优先）
- 或更常被引的（核心文献优先）
- 或按时间顺序（演化脉络）

### 6.2 每篇正文抽取模板

**对每篇论文，只回答以下内容**：

#### 3A 假设/成立条件
- 核心假设是什么
- 现实中是否难满足
- 违反假设的后果

#### 3B 好处/贡献
- 核心贡献1-3点
- 相比之前工作的改进

#### 3C 关键机制/关键公式项
- 仅3-5个关键点
- 不需要全部公式

#### 3D 缺点/局限
- 主要缺点
- 缺点的来源是什么（理论/实现/数据）

#### 额外检查
- **适用/不适用场景结论**
- **Examples检查**：是否可能只展示成功案例？哪些失败情形未覆盖？

### 6.3 每派总结包

**每个派别包含**：
- 发展脉络（时间线）
- 理论依据（数学/逻辑基础）
- 主要成果（代表性结果）
- 缺点来源（为什么有这些缺点）
- 适用/不适用结论

---

## Step 8: 最终收束

### 8.1 机会矩阵/对照表

**行**：派别/方法
**列**：
- 优点
- 缺点
- 关键假设
- 适用场景
- 不适用场景
- 实施成本
- 可改良点

### 8.2 基于场景的推荐路线

**给出2-3条路线，每条包含**：
- 路线名称
- 推荐派别组合
- 理由（为什么适合该场景）
- 风险/注意事项

### 8.3 创新切入点清单

**必须来自"优缺点 × 场景需求"的对齐**：

每条包含：
- **要改什么**：具体改动点
- **为什么**：解决什么痛点
- **可能代价/风险**：权衡
- **验证方式**：如何验证有效

### 8.4 下一步行动清单

- 需要检索补齐的内容
- 必读论文顺序
- 要验证的关键假设与指标
