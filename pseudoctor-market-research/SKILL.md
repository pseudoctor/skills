## 研究方法论协议

### 原则层级（严格遵循）

#### Priority 1: 绝对正确性

- ✗ 绝不编造任何信息
- ✗ 用置信度标记不确定性
- ✓ 关键论断需至少3个独立来源
- ✓ 不确定时明确说"我无法验证此点"

#### Priority 2: 完美理解

- ✓ 检测歧义、矛盾和错误前提
- ✓ 研究前澄清问题
- ✓ 通过互动确认理解

#### Priority 3: 全面质量

- ✓ 提供完整、具体、可操作的信息
- ✓ 深度优于广度
- ✓ 展示完整的证据推理链

### 五阶段研究协议

#### PHASE 1: 查询验证 (QUERY VALIDATION)

**1.1 分析用户问题**

检查以下维度：
- ✅ 清晰度和具体性
- ✅ 逻辑一致性
- ✅ 上下文完整性
- ✅ 可行性评估

**1.2 问题分类**

- 行业分析
- 公司研究
- 产品分析
- 竞争情报
- 投资研究
- 市场进入

**1.3 澄清机制**

如发现问题，使用以下模板主动澄清：

```
📋 需要澄清

检测到的问题：
[具体问题描述]

请确认或提供：
1. [需要澄清的具体点]
2. [需要的额外上下文]

示例回答格式：
"我想了解[明确主题]在[具体语境]中的[具体方面]"
```

**1.4 研究范围确认**

明确以下要素：
- 地理范围（全球/区域/国家）
- 时间范围（历史/当前/预测）
- 细分市场（整体/特定细分）
- 分析深度（概览/深度/专项）

#### PHASE 2: 深度分析 (DEEP ANALYSIS)

**2.1 第一原理分解**

将研究问题分解为可验证的假设：
```
核心问题
├── 子问题1
│   ├── 可验证假设A
│   └── 可验证假设B
├── 子问题2
│   ├── 可验证假设C
│   └── 可验证假设D
└── 子问题3
    ├── 可验证假设E
    └── 可验证假设F
```

**2.2 证据需求规划**

| 假设 | 证据类型 | 来源层级 | 数量要求 |
|------|----------|----------|----------|
| 假设A | 市场数据 | Tier 1 | ≥3个 |
| 假设B | 财务数据 | Tier 1 | ≥2个 |
| 假设C | 消费者调研 | Tier 2 | ≥1个 |

**2.3 搜索策略规划**

设计渐进式搜索序列：
1. 宽泛搜索建立基础理解
2. 针对性搜索验证假设
3. 反向搜索寻找矛盾证据
4. 最新数据搜索确保时效性

#### PHASE 3: 系统研究 (SYSTEMATIC RESEARCH)

**3.1 渐进式搜索执行**

##### 搜索1：核心信息
```
"[主题] 市场规模 增长 2024"
"[主题] 权威报告 研究机构"
```
- 目的：建立基础理解
- 来源偏好：Tier 1（官方、学术）

##### 搜索2：学术验证
```
"[主题] 研究 paper 同行评议"
"[假设] 学术验证 证据"
```
- 目的：验证关键假设
- 来源偏好：Tier 1（学术期刊）

##### 搜索3：竞争情报
```
"[竞争对手] 市场份额 战略 动向"
"[主题] 竞争格局 分析"
```
- 目的：了解竞争态势
- 来源偏好：Tier 1-2

##### 搜索4：事实核查
```
"[具体论断] fact check 核实"
"[数据] 验证 来源"
```
- 目的：交叉验证关键数据
- 来源偏好：多源印证

##### 搜索5：最新动态
```
"[主题] 2024 2025 最新 趋势"
"[主题] recent developments"
```
- 目的：确保信息时效性
- 来源偏好：新闻、官方发布

##### 搜索6：矛盾观点
```
"[主题] 批评 问题 局限 风险"
"[主流观点] 反驳 质疑"
```
- 目的：识别潜在风险
- 来源偏好：多元观点

**3.2 来源评估体系**

| Tier | 类型 | 信任度 | 使用权重 |
|------|------|--------|----------|
| **Tier 1** | 官方机构、学术期刊、政府统计 | ⭐⭐⭐⭐⭐ | 主要来源 |
| **Tier 2** | 既定媒体、行业专家、知名咨询 | ⭐⭐⭐⭐ | 辅助印证 |
| **Tier 3** | 多个相互印证的次级来源 | ⭐⭐⭐ | 补充参考 |
| **Reject** | 无来源、被证伪、明显偏见 | ⭐ | 不使用 |

**3.3 数据记录规范**

对每条记录：
```markdown
**[数据/论断]**
- 内容: [具体内容]
- 来源: [完整引用]
- 日期: [数据日期]
- 置信度: [高/中/低]
- 备注: [附加说明]
```

#### PHASE 4: 协作增强 (COLLABORATIVE ENHANCEMENT)

**4.1 缺口识别**

识别以下类型缺口：
- 📍 数据缺口（无法找到的关键数据）
- 📍 时效缺口（最新数据缺失）
- 📍 验证缺口（无法交叉验证）
- 📍 深度缺口（信息不够详细）

**4.2 协作请求模板**

如遇到关键信息缺口，使用以下模板：

```
🤝 需要您的协助

研究主题：[当前研究主题]

已验证信息：
✓ [已确认的关键信息1] [置信度:95%]
✓ [已确认的关键信息2] [置信度:90%]
✓ [已确认的关键信息3] [置信度:85%]

信息缺口：
○ [缺失的关键信息1] - [为什么重要]
○ [缺失的关键信息2] - [为什么重要]

如您能提供以下资源，可显著提升分析质量：
- [具体资源类型1，如：公司内部报告]
- [具体资源类型2，如：行业专家访谈]

基于当前信息，我可以确认：
[基于已有信息的部分结论]

如无法提供额外资源，我将：
[替代方案，如：基于公开信息给出带置信度标记的结论]
```

**4.3 假设管理**

对无法验证的假设：
- 明确标记为"假设"
- 说明支持证据的局限性
- 提供验证路径建议

#### PHASE 5: 综合输出 (SYNTHESIS & OUTPUT)

**5.1 信息综合原则**

- 整合而非罗列
- 逻辑链条完整展示
- 不确定因素明确标记
- 置信度清晰标注

**5.2 置信度标注规范**

```markdown
**[数据/论断]** [置信度: XX%]

- 内容: [具体内容]
- 来源: [完整引用]
- 日期: [数据日期]
- 验证状态: [已交叉验证/单一来源/无法验证]
```

置信度等级：
- **95%+**: 多个权威来源一致确认
- **85-94%**: 多个可信来源基本一致
- **70-84%**: 单一权威来源或多个来源部分一致
- **50-69%**: 单一次级来源或信息有限
- **<50%**: 信息不足，仅作为参考

**5.3 质量自检**

在输出前执行：
- [ ] 逻辑一致性检查
- [ ] 数据交叉验证
- [ ] 来源完整性确认
- [ ] 置信度合理性评估

## 分析框架详解

### 第一部分：执行摘要 (Executive Summary)

**1.1 内容要求**
- 2-3页，高度概括核心发现
- 放在报告最开头
- 在所有其他章节完成后撰写

**1.2 必须包含的要素**

##### 市场快照
```markdown
### 📊 市场快照 Market Snapshot

| 指标 | 数据 |
|------|------|
| 市场规模 | [当前市场规模] |
| 增长率 | [CAGR] |
| 关键驱动 | [1-3个驱动因素] |
| 核心挑战 | [1-3个挑战] |
| 最大机会 | [最关键机会] |
```

##### 核心发现
- 3-5个最重要的发现
- 每个发现有数据支撑
- 标注置信度

##### 战略建议
- 3-5个可操作的战略建议
- 按优先级排序
- 包含预期影响

### 第二部分：核心产品与产品线分析

**2.1 核心产品定义**
- 产品/服务的主要功能与特性
- 目标客户群体及其核心价值主张
- 解决的核心问题

**2.2 产品线矩阵分析**

*广度分析：*
- 覆盖的细分市场清单
- 各细分市场的产品覆盖度
- 市场空白点识别

*深度分析：*
- 各细分市场的产品型号/规格
- 产品差异化特征
- 价格带分布

*关联性分析：*
- 产品线协同效应图
- 潜在冲突识别
- 组合销售机会

**2.3 产品竞争力评估**

*USP分析：*
- 独特销售主张清单
- 与竞品差异化对比
- 护城河深度评估

*技术与创新：*
- 当前技术水平定位
- 研发投入分析
- 专利/技术壁垒
- 未来创新方向

*成本与定价：*
- 成本结构分解
- 定价策略分析
- 价格竞争力评估

*质量与品牌：*
- 质量控制体系
- 消费者品牌感知
- 品牌资产价值

**2.4 产品生命周期**
- 主要产品所处阶段判定
- 生命周期阶段矩阵
- 产品组合健康度评分
- 新老产品迭代节奏

### 第三部分：市场宏观环境与现状

**3.1 市场规模与增长**

*当前市场容量：*
- 市场规模（价值量/数量）
- 历史增长轨迹（5-10年）
- 市场增长驱动因素
- 市场增长制约因素

*TAM/SAM/SOM分析：*
- TAM（总可寻址市场）测算
- SAM（可服务市场）定义
- SOM（可获得市场）估算
- 市场渗透率分析

*未来预测：*
- 3-5年市场规模预测
- CAGR预测
- 关键假设说明
- 情景分析（乐观/基准/悲观）

**3.2 宏观环境分析 (PESTEL)**

*Political（政治/政策）：*
- 产业政策支持度
- 法规变化影响
- 税收政策影响
- 贸易关系影响
- 政府采购导向

*Economic（经济）：*
- 宏观经济周期位置
- 居民收入水平
- 消费信心指数
- 通胀影响
- 汇率影响（如涉及进出口）

*Social（社会）：*
- 人口结构变化
- 消费习惯变迁
- 文化价值观演变
- 生活方式变化
- 教育水平影响

*Technological（技术）：*
- 相关技术突破
- 技术应用趋势
- 研发热点
- 技术壁垒
- 数字化影响

*Environmental（环境）：*
- 环保法规影响
- 可持续发展要求
- 气候变化影响
- 碳中和目标影响

*Legal（法律）：*
- 行业准入要求
- 知识产权保护
- 劳动法影响
- 消费者权益保护
- 反垄断法规

**3.3 竞争格局分析**

*主要竞争者识别：*
- 核心竞争对手清单
- 各竞争对手市场份额
- 战略动向追踪
- 财务表现对比

*市场集中度分析：*
- CR5/CR10集中度指标
- HHI指数
- 市场结构判定（碎片化/寡头垄断）

*竞争强度评估：*
- 价格竞争激烈程度
- 产品创新竞争
- 营销/品牌竞争
- 渠道争夺程度

*波特五力模型（必须包含）：*
- 现有竞争者竞争强度
- 新进入者威胁
- 供应商议价能力
- 购买者议价能力
- 替代品威胁
- 综合吸引力评估

### 第四部分：目标市场表现与消费者洞察

**4.1 市场地位与表现**

*市场份额分析：*
- 整体市场份额
- 细分市场份额
- 市场份额变化趋势
- 相对市场地位

*品牌健康度：*
- 品牌知名度
- 品牌美誉度
- 品牌忠诚度
- 品牌联想分析

*财务表现：*
- 营收增长趋势
- 盈利能力分析
- 资产效率
- 与行业/竞品对比

**4.2 消费者深度洞察**

*目标群体画像：*
- 人口统计学特征
- 地理分布特征
- 心理特征
- 行为模式特征

*购买决策旅程：*
- 认知阶段：信息获取渠道
- 考虑阶段：评估比较标准
- 购买阶段：决策触发因素
- 购后阶段：满意度与复购

*核心诉求与痛点：*
- 功能性需求清单
- 情感性需求清单
- 未满足需求识别
- 痛点优先级排序

*品牌互动与触点：*
- 关键接触渠道
- 体验关键时刻
- 流失原因分析
- 增强机会点

### 第五部分：销售渠道与分销体系剖析（核心重点）

**5.1 渠道全景图**

*线上渠道：*
- 自营电商：官网、APP、小程序
- 平台电商：天猫、京东、拼多多
- 社交电商：微信、抖音、快手
- 直播电商：平台直播、品牌自播

*线下渠道：*
- 直营渠道：专卖店、专柜
- 加盟渠道：加盟店、连锁
- 经销商/代理商体系
- 第三方零售：商超、便利店

*O2O融合：*
- 线上线下联动模式
- 全渠道体验设计
- 数据打通程度
- 效果评估指标

**5.2 线下渠道深度剖析**

*市场层级覆盖：*
- 一线城市：渗透率、布局策略
- 二线城市：渗透率、布局策略
- 三线城市：渗透率、布局策略
- 四五线及以下：渗透率、布局策略
- 县域/乡镇市场：特殊考量

*渠道结构与网络：*
- 分销层级（一级/二级/三级）
- 经销商数量与质量
- 门店类型分布
- 选址策略分析

*渠道管理体系：*
- 经销商选择标准
- 经销商评估机制
- 激励政策体系
- 淘汰机制
- 价格管控体系
- 窜货管理机制

*渠道营销支持：*
- 终端促销政策
- 物料支持体系
- 导购培训体系
- 区域市场活动支持

*物流与供应链：*
- 仓储布局策略
- 配送效率指标
- 供应链响应速度
- 成本控制措施

**5.3 渠道效率与挑战**

*关键指标评估：*
- 覆盖率（地理/人口）
- 动销率（售点活跃度）
- 坪效（每平米销售额）
