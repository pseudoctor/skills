# 多论文文献综述与学术地图 Skill

该 skill 用于把一组已经给定的论文材料，转化为可审计的结构化综述结果。重点不是复述单篇论文，而是识别领域中的共同假设、主要分歧、概念谱系、方法格局、研究空白与当前共识。

输出目标：
- 列清论文总表，而不是只说“看过了”
- 画出学术地图，而不是散乱总结
- 抓出作者之间的直接冲突，并解释冲突为何产生
- 梳理 3 个核心概念的谱系，而不是堆概念定义
- 提炼未解问题、方法偏好、隐含假设
- 最终压缩成短综述、知识图谱和面向外行的解释
- 如用户需要，可进一步压缩成带脚注的综述短稿

输出语言：
- 默认使用简体中文
- 论文标题保留原文
- 关键概念优先保留原文，可在括号内补中文或英文释义

语言适配：
- 支持中文论文
- 支持英文论文
- 支持中英文混合语料
- 默认根据用户提问语言输出；若用户用中文提问，默认中文输出；若用户用英文提问，默认英文输出；若不明确，可用双语字段名
- 无论输出语言为何，论文标题、作者名、原始概念名优先保留原文

## 何时使用

当用户出现以下诉求时使用：
- “根据这些论文写文献综述”
- “帮我把这些论文的分歧理出来”
- “给我做个学术地图/知识图谱”
- “找出研究空白、争议区、未解之谜”
- “梳理这个领域的核心概念演化”
- “比较这些论文的方法优劣”
- “把这个领域在 5 分钟内讲给外行听”
- “中英文论文混在一起，帮我做综述”

不要在以下场景强行使用：
- 用户只想精读单篇论文
- 用户需要先全网搜文献、补 DOI、查真实引用量
- 用户只要改写已有综述段落，不需要结构化分析

## 触发描述变体

以下是可直接复用的两版触发描述，可按宿主模型风格选用。主 skill 仍以 YAML frontmatter 中的 `description` 为准，这里提供增强版措辞。

### Claude 风格触发描述

当用户给出一组论文，希望你不要逐篇摘要，而是把整组文献的结构关系讲清楚时，使用这个 skill。它适合处理文献综述、学术地图、观点冲突、概念谱系、研究空白、方法比较、隐含假设和面向外行的领域解释。尤其当用户说“帮我写综述”“比较这些论文”“找出它们在吵什么”“给我一个知识图谱”“告诉我这个领域到底搞明白了什么”时，应优先触发。只基于用户提供的论文和文本材料工作，不擅自补全外部事实；若证据不足，明确降级并标注不确定性。

### Gemini 风格触发描述

Use this skill whenever the user provides a set of papers and wants a structured, cross-paper synthesis rather than isolated summaries. This includes literature reviews, academic maps, contradiction tables, concept lineages, unresolved questions, method comparisons, hidden assumptions, knowledge graphs, and plain-language field briefings. Trigger even if the user does not explicitly say "literature review" but is clearly asking to compare papers, map a field, surface debates, identify gaps, or explain what the research community knows versus what remains unsettled. Work only from the supplied papers and excerpts. If the corpus is incomplete, reduce certainty explicitly instead of guessing.

## 核心原则

### 1. 强证据绑定

关键判断必须尽量绑定到用户提供的论文原文、摘要、引言、结论、方法部分或元数据。以下内容不能凭印象断言：
- 直接冲突
- 学界共识
- 引用最多或最核心的概念
- 方法最弱的论文
- 研究空白为何尚未解决

若证据不足，必须明确写：
- `[证据不足]`
- `[需全文核验]`
- `[当前材料无法判断]`

若处理中英文混合语料：
- 证据引用保持原文语言
- 概念解释可用用户语言补充
- 不要擅自把英文概念硬翻成不常用中文术语，也不要把中文政治/哲学概念硬译成失真的英文短语

### 2. 面向“论文集合”而不是单篇

你的任务是比较、归类、抽象、对照，不是按论文一篇一篇做流水账摘要。

### 3. 先盘点材料层级，再决定输出力度

必须先判断用户提供的是全文、摘要+引言+结论、还是只有题录。不同材料层级决定能输出到什么深度。

### 3.5 起步阶段必须先看摘要、导言和结论

除非用户当前真的只提供了题名或纯题录，否则不能在起步阶段只依赖题名做分析或分组。

硬规则：
- 只要拿得到摘要，先读摘要
- 只要拿得到引言，先读引言中的问题意识和研究对象
- 只要拿得到结论，先读结论中的最终判断和贡献表达

中英文对应结构标记：
- 中文：摘要、内容提要、提要、关键词、引言、问题提出、研究背景、结论、结语、结束语
- 英文：Abstract, Keywords, Introduction, Background, Conclusion, Conclusions, Discussion, Final Remarks

起步阶段最少要优先提取：
- 论文想回答什么问题
- 作者最终主张什么
- 研究对象、场景或材料是什么
- 论文大致采用什么方法或论证方式

只有完成这一步，才允许开始：
- 判断哪些论文更重要
- 形成核心池
- 画学术地图
- 写冲突矩阵

如果当前只能读题名，必须明确说明这是临时降级，而不是默认流程。

### 3.6 如果输入是 PDF，优先抽取摘要、导言和结论

如果用户提供的是 PDF 文献，默认先做轻量信息抽取，而不是直接凭文件名或题名开始分析。

PDF 起步抽取顺序：
1. 题名页：确认标题、作者、年份、期刊/来源
2. 摘要页：提取摘要/内容提要/提要/关键词
3. 导言部分：提取“引言”“问题提出”“研究背景”等段落
4. 结论部分：提取“结论”“结语”“结束语”等段落
5. 如有必要，再补正文关键段落

如果是英文论文，对应抽取：
1. Title page
2. Abstract / Keywords
3. Introduction / Background
4. Conclusion / Conclusions / Discussion / Final Remarks
5. Key body paragraphs if needed

优先目标不是一次读完整篇，而是先拿到可用于判断重要性的最小信息包。

最小信息包包括：
- 标题
- 作者
- 年份
- 摘要主张
- 导言问题
- 结论判断
- 方法/论证方式

如果是中英文混合语料，最小信息包还应补：
- 论文原文语言
- 关键概念原文写法

如果 PDF 文本抽取效果不好，不要立刻退回纯题名分析。先尝试：
- 改用题名页和摘要页文字
- 改用引言、结论所在页面片段
- 对少量候选核心文献做人工校准式抽取

只有在确实拿不到摘要、导言、结论时，才降级为题名级预分析。

### 4. 优先给结构，不先给散文

优先输出表格、矩阵、分组、谱系、提纲式知识图谱。只有在最后一部分再压缩成 400 字综述和 5 分钟讲解。

## 输入模式判断

收到材料后，先判定属于哪一档：

### Mode A: Full Corpus

适用于：
- 大部分论文有全文
- 至少能读到摘要、引言、结论、方法和部分正文

处理规则：
- 可以执行完整 9 模块输出
- 可以做相对强的冲突判断、方法评估和空白识别

### Mode B: Partial Corpus

适用于：
- 论文只有摘要、引言、结论，或全文覆盖不完整

处理规则：
- 可以输出学术地图、初步冲突矩阵、概念谱系、方法分组、初步研究空白
- 所有高风险判断必须标 `[需全文核验]`
- 不能把初步归纳写成确定共识
- 若中英文论文混合，分组时优先按论点和问题意识，而不是按语言分组

### Mode C: Bibliography Only

适用于：
- 只有标题、作者、年份、少量摘要或题录

处理规则：
- 先输出“可分析框架 + 缺失材料清单”
- 不直接产出高确定性综述
- 可以给出待提取信息模板，指导用户补材料

### Mode D: Title-Level Pre-Mapping

适用于：
- 用户给了大量 PDF 或题录
- 当前只能稳定读取题名、作者、年份，或只能做题名级预分组

处理规则：
- 只能输出“题名级预分析”，不能伪装成全文级判断
- 可以输出：预学术地图、预分组、预冲突线、核心池建议、补料优先级
- 不可以输出：高确定性共识、直接冲突定论、方法强弱定论
- 所有判断默认标注 `[题名级预分析]` 或 `[需摘要/引言/结论核验]`

当语料规模很大而材料层级很浅时，优先进入 Mode D，而不是硬做完整 9 模块。

注意：
- Mode D 只是兜底，不是首选
- 只要可以补抽摘要、引言、结论，就应尽快退出 Mode D
- 不要把“先按题名粗分组”当作正式起步方法

## 最低输入要求

最低要求：
- 每篇论文至少提供：标题、作者、年份
- 并且至少有一段可分析文本，哪怕只是摘要

推荐要求：
- 摘要 + 引言 + 结论

最佳要求：
- 全文，或至少有方法与讨论部分

## 大语料优先规则

当论文数量很多时，不要一上来试图逐篇做完整分析。

推荐阈值：
- 1-15 篇：可直接进入完整结构分析
- 15-40 篇：先做材料盘点，再决定是否需要缩小核心池
- 40 篇以上：默认先做“核心池收缩”，不要直接写完整综述

### 核心池收缩

当论文超过 40 篇时，先从全部语料里筛出一个“核心池”，再对核心池做强分析。

核心池不是固定数量任务，而是内容驱动任务。不要预设一定要筛到 8 篇、12 篇或 20 篇；应根据用户提供的文献内容、领域内部结构和分歧密度决定保留规模。

核心池优先保留：
- 概念界定类
- 理论/思想体系类
- 思想源流类
- 典型实践场景类
- 综述/述评类
- 关键扩展类（如全球治理、跨区域外延）

筛选标准优先看：
- 是否代表一个关键论证方向
- 是否构成主要分歧线的一端
- 是否被多篇论文反复回应或承接
- 是否提供独特的方法、对象或场景
- 是否对领域整体结构有枢纽作用
- 是否只是重复已有论文而新增量很少

### 核心池评分表

为了让筛选更稳定，建议先给候选文献做一个“核心池评分表”，再决定是否纳入核心池。

默认评分维度：
- `代表性`：是否能代表一个关键论证方向或典型立场
- `分歧性`：是否处于主要争议线的一端，或能帮助构造冲突矩阵
- `枢纽性`：是否被多篇文献承接、回应、转化，或位于领域结构中心
- `方法独特性`：是否提供独特的方法、场景、材料或研究对象
- `重复度`：与其他候选文献相比是否只是重复表达，新增量是否有限

推荐评分方式：
- `高 / 中 / 低`
- 或 `1-5 分`

使用规则：
- `代表性`、`分歧性`、`枢纽性` 越高，越优先进入核心池
- `方法独特性` 高的论文，即使不是主流，也可能保留为补充核心文献
- `重复度` 高的论文，除非它是经典表述或权威综述，否则优先降为边界文献

评分解释句式模板：
- `代表性`：
  - “该文能够较集中地代表本领域的一条核心论证路径，因此代表性较高。”
  - “该文虽然论述完整，但所代表的立场与已有文献高度重合，因此代表性有限。”
- `分歧性`：
  - “该文处于主要争议线的一端，对构造领域内部的核心分歧具有较强价值。”
  - “该文更多是在既有立场内部做补充说明，对主要分歧线的推进相对有限。”
- `枢纽性`：
  - “该文在概念界定、理论转接或场景扩展上具有明显枢纽作用，能够连接多组文献。”
  - “该文在本组内部具有一定参考价值，但对整体领域结构的连接作用较弱。”
- `方法独特性`：
  - “该文在研究对象、案例选择或分析路径上提供了不同于主流文献的独特增量。”
  - “该文的方法路径与现有研究大体一致，方法上的独特贡献相对有限。”
- `重复度`：
  - “该文与已有核心文献在问题意识和结论表达上重复度较高，新增量相对有限。”
  - “该文虽与既有研究存在主题重合，但在论证重点或材料使用上仍保有一定差异。”

建议：
- 给出评分后，尽量为每篇论文至少补 1 句解释
- 不要只写“高/中/低”而不给理由
- 解释句应保持学术语体，避免口语化评价

不要机械汇总分数后自动决定去留。评分表是辅助判断工具，最终仍要结合：
- 用户目标
- 语料结构
- 当前分析任务（做地图、做冲突、做综述、做方法比较）

推荐顺序：
1. 先做最小阅读卡
2. 再做核心池评分表
3. 最后确定核心池和边界文献

核心池的筛选依据，优先级必须是：
1. 摘要
2. 引言
3. 结论
4. 正文关键段落
5. 最后才是题名

禁止做法：
- 只看题名就断定“核心文献”
- 只按关键词命中次数决定重要性
- 只按派别平均抽样，不看论文实际主张
- 为了凑数量硬把边缘文献塞进核心池
- 不看文献内容就直接把核心池缩到某个固定篇数

推荐做法：
- 先为每篇论文做一个最小阅读卡：题名 + 摘要主张 + 引言问题 + 结论判断 + 方法/论证方式
- 再基于这些阅读卡筛核心池

如果输入主要是 PDF，大语料缩池的默认动作应是：
- 先批量抽取候选论文的题名页、摘要、导言、结论
- 再基于抽取结果缩池

不要跳过抽取步骤直接按文件名缩池。

输出时明确写：
- 核心池规模为什么是现在这个大小
- 为什么这些论文被选入核心池
- 哪些文献暂时作为边界文献保留
- 核心池之外的判断都只能作为弱结论

## 证据层级

输出前，先判断你当前引用的证据属于哪一级。

- `题名级`：只能从标题、作者、年份推断主题方向
- `摘要级`：可提炼核心命题，但仍可能遗漏限定条件
- `引言级`：可判断问题意识、研究对象、部分理论定位
- `结论级`：可判断作者的最终主张与贡献表达
- `正文片段级`：可支持局部论证、概念定义、分歧点
- `全文级`：可做较强的冲突判断、方法评估与空白提炼

语言处理规则：
- 中文论文：优先保留中文标题、概念、引文
- 英文论文：优先保留英文标题、概念、引文
- 混合语料：允许中文解释 + 英文术语，或英文解释 + 中文概念原文
- 不要因为论文语言不同而降低其在核心池中的优先级

规则：
- 若结论只基于题名，不得写成确定判断
- 若结论只基于摘要/引言/结论，不得冒充全文级把握
- 若要判断“直接冲突”，至少应有正文片段级或结论级的明确对立证据

## 标准工作流

### 第 0 步：材料盘点

先输出：
- 当前论文总数
- 每篇论文的可用材料层级
- 哪些论文信息不完整
- 本轮能够完成哪些模块
- 哪些模块必须降级

不要跳过这一步。

如果是大语料，材料盘点后优先判断：
- 是否进入核心池收缩
- 是否只能做题名级预分析
- 哪些论文最值得先抽取摘要/引言/结论

默认动作不是“先分组”，而是“先抽摘要/引言/结论”。

如果输入是 PDF，材料盘点后还应明确：
- 当前是否具备稳定的 PDF 文本抽取条件
- 哪些论文已成功抽到摘要/导言/结论
- 哪些论文需要人工校准或暂时降级

### 第 1 步：论文总表

为所有论文建立总表，最少包含：
- 作者
- 年份
- 标题
- 一句话核心观点
- 方法类型
- 证据状态

若核心观点无法从当前材料唯一推出，写 `[无法从当前材料唯一推出]`。

如果论文数量很多，先建立“最小阅读卡总表”，至少包含：
- 作者
- 年份
- 标题
- 原文语言
- 摘要中的核心主张
- 引言中的问题意识
- 结论中的最终判断
- 方法/论证方式
- 当前证据层级

没有摘要、引言、结论支持时，不要急于写“一句话核心观点”。

如果来自 PDF 抽取，优先记录每一列信息的来源位置，例如：
- 摘要页
- 引言第一页
- 结论段
- 正文片段

### 第 2 步：学术地图

按“共同假设”或“共同问题意识”分组，而不是按年份机械分组。

分组前必须优先比较：
- 摘要里的核心命题
- 引言里的问题意识
- 结论里的最终主张

如果是中英文混合语料：
- 先统一概念对应关系，再分组
- 同义概念可写成 `原文概念 A / Original Term B`
- 不要把语言差异误判成理论分歧

不要只按题名相似度或关键词相似度分组。

至少输出：
- 每个组的名称
- 该组的共同假设
- 代表论文
- 与其他组的主要分歧线

此处不要提前写成长综述。

### 第 3 步：冲突矩阵

找出作者观点直接冲突的地方。每个冲突点最少包含：
- 冲突议题
- 立场 A
- 立场 B
- 对应论文
- 分歧原因
- 证据位置

分歧原因优先从以下角度分析：
- 方法不同
- 数据或材料不同
- 研究对象或样本不同
- 时代背景不同
- 概念定义不同

如果证据不足以证明“直接冲突”，降级为“解释性张力”或“潜在分歧线”。

区分标准：
- `直接冲突`：双方对同一议题给出明确对立判断
- `解释性张力`：研究重心、分析层级、价值排序不同，但未必直接互斥
- `潜在分歧线`：目前只能从题名、摘要或有限片段看出可能存在分歧

### 第 4 步：核心概念谱系

找出当前论文集合中反复出现、且支撑领域论证的 3 个核心概念。

注意：
- 若没有真实引用计数，不能说“被引用最多”
- 必须改写为“在当前论文集合中高频出现的 3 个核心概念”

每个概念至少输出：
- 概念名称
- 奠基者/早期代表
- 主要挑战者
- 关键改进者
- 当前共识（如有）
- 仍未解决的问题

若概念跨中英文论文出现，建议格式：
- `正确义利观 / the principle of pursuing the greater good and shared interests`
- `人类命运共同体 / community with a shared future for mankind`

优先保留学界更常用的原文形式。

### 第 5 步：研究空白

基于全部论文提炼 5 个尚未被充分解答的问题。

每个空白点至少输出：
- 问题本身
- 为什么一直没解决
- 哪篇论文最接近答案
- 还需要什么新方法或新材料

不要把“作者在结论里建议未来研究”原样抄成空白点，要做跨论文综合。

### 第 6 步：方法比较

按方法类型分组，例如：
- 问卷调查
- 实验
- 模拟
- 元分析
- 案例研究
- 规范分析
- 文本阐释
- 历史文献分析

如果领域主要是规范理论、思想史、政策文本或外交理念研究，不要硬套经验主义方法分类。优先使用：
- 规范分析
- 概念分析
- 思想史/概念史
- 政策文本分析
- 个案分析
- 比较案例

至少输出：
- 方法类别
- 代表论文
- 常用于回答什么问题
- 优势
- 盲区
- 该领域最常用的方法及原因
- 被用得太少的方法及原因
- 方法最薄弱的论文及原因

评价“方法最薄弱”时必须说清标准，避免情绪化用语。可用标准包括：
- 样本过窄
- 识别策略弱
- 论证链条不闭合
- 证据和结论不匹配
- 缺乏比较或反例

### 第 7 步：压缩综述

在前述结构分析完成后，输出一段不超过 400 字的综述。

必须覆盖：
- 当前较稳的共识
- 仍在争论的问题
- 哪些结论有较强证据支撑
- 最关键的未解之谜

不要复述单篇论文，不要写空泛套话。

### 第 7A 步：综述短稿（可选）

如果用户要求“更像正式论文”“压缩成短稿”“带引文/脚注”“中文核心期刊风格”，则在完成第 7 步后，继续输出一版综述短稿。

默认要求：
- 长度：800-1200 字
- 形式：连续正文，不写成表格或提纲
- 功能：像文献综述小节，而不是口语化总结

如果用户明确要求脚注，使用“正文上标序号 + 页下注释”的写法。

### 第 7B 步：中文核心期刊风格脚注版（可选）

如果用户要求“中文核心期刊风格脚注”“脚注版综述”，执行以下规则：

- 正文保持学术论文语体，不要口语化
- 引文使用上标序号，如 `①` `②`
- 页下注释放在正文之后，按出现顺序排列
- 同一条脚注可并列列出 2-3 篇相关文献
- 优先使用已从材料中确认的作者、篇名、期刊、年份、期次、页码
- 若卷期页码未核实，不得编造；可退而使用“作者：《篇名》”或“作者：《篇名》，年份”这种保守写法
- 若引文仅基于摘要、引言、结论，不要在脚注中伪装成精确页码定位

脚注版短稿的作用：
- 把前面的结构化分析压缩成可直接放进论文中的综述性段落
- 保留最关键的分歧、共识与空白
- 让用户能继续扩展成正式论文综述部分

### 第 8 步：隐含假设

列出多数论文默认接受、但很少被正面验证或解释的假设。

每个假设至少输出：
- 假设内容
- 最依赖它的 1-2 篇论文
- 为什么这个假设常被默认
- 如果这个假设错了，领域会怎样

### 第 9 步：知识图谱 + 5 分钟讲解

输出两部分。

第一部分是结构化知识图谱，至少包含：
- 核心命题
- 3-5 个支撑柱
- 2-3 个争议区
- 1-2 个前沿问题
- 新手必读的 3 篇论文与理由

第二部分是面向聪明外行的 5 分钟讲解，固定输出 3 句话：
- 一句话总结：这个领域到底搞明白了什么
- 一句大实话：它最大的未知是什么
- 一个现实影响：这件事对现实最实在的影响是什么

要求：
- 不要行话堆砌
- 不要官腔
- 不要假装确定

## 输出顺序

默认按以下顺序输出：
1. 材料盘点
2. 论文总表
3. 学术地图
4. 冲突矩阵
5. 核心概念谱系
6. 研究空白
7. 方法比较
8. 压缩综述
9. 隐含假设
10. 知识图谱
11. 5 分钟讲解

如果用户额外要求“综述短稿”或“脚注版综述”，在第 8 项之后追加：
12. 综述短稿（可选）
13. 脚注版综述短稿（可选）

如果用户只点名某一模块，可以只做该模块，但仍要先声明材料层级和证据边界。

## 停顿与确认规则

当论文数很多或材料不齐时，不要一次性把所有内容倾倒给用户。

推荐停顿点：
- 完成“材料盘点 + 论文总表”后
- 完成“学术地图 + 冲突矩阵”后
- 完成“概念谱系 + 空白 + 方法比较”后
- 最后再做压缩综述与外行讲解

如果用户明确要求“一次给完”，可以连续输出，但仍要保持结构化。

## 失败与降级策略

### 如果没有全文

- 冲突矩阵中对“分歧原因”的判断标 `[需全文核验]`
- 方法优劣判断标 `[基于摘要/结论的初步判断]`
- 共识判断标 `[当前材料下的暂定共识]`
- 若只掌握标题或摘要，不要直接给出“直接冲突”，先降级为“潜在分歧线”

### 如果只有题录

只输出：
- 缺失材料清单
- 建议优先补哪几篇论文
- 建议优先提取哪些字段
- 可直接复用的信息提取模板

### 如果只能稳定提取题名页和摘要页

优先输出：
- 核心池建议
- 摘要级学术地图
- 初步概念簇
- 初步争议线
- 下一轮最值得补读的论文

如果已经拿到摘要，不要再称之为“题名级预分析”，应升级为“摘要级初步分析”。

如果已经拿到导言或结论，再进一步升级为“引言/结论级初步分析”。

### 如果论文彼此不在同一议题

先指出语料异质性太强，再尝试按子议题拆分，不要硬凑成一个统一领域。

## 风格要求

- 始终简洁、清晰、少套话
- 优先表格、矩阵、提纲
- 用“这组论文共同假设……”“冲突集中在……”“较稳的结论是……”这类句式
- 避免“本文认为”“笔者认为”等论文腔
- 中英文混合时，不要因为翻译追求整齐而牺牲概念准确性

## 禁止事项

- 不要编造论文结论
- 不要把高频出现误写成“高被引”
- 不要把“作者没讨论”自动等同于“领域空白”
- 不要在证据不足时装作很确定
- 不要用侮辱性措辞评价方法，只说明其薄弱点
